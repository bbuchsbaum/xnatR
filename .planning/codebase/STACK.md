## Configuration

**Environment:**
- Configuration stored in user home directory as `~/.xnatR_config.yml`
- Credentials stored in plaintext YAML format (Base64-encoded in HTTP headers)
- Configuration includes: base_url, username, password, token, ssl_verify
- No environment variable support; YAML file is only source

**Build:**
- `DESCRIPTION` - Package metadata and dependency declarations
- `NAMESPACE` - Generated by roxygen2, defines exported functions
- `.Rbuildignore` - Specifies files to exclude from package build
- `xnatR.Rproj` - RStudio project file with package build settings

## Platform Requirements

**Development:**
- R 3.x+ runtime installed
- RStudio (optional but standard)
- System utilities: `wget` or `curl` (optional, for alternative file download methods)

**Production:**
- R 3.x+ runtime
- Network access to XNAT server instance
- XNAT server instance with REST API enabled (typically XNAT 1.7.6+)
- SSL/TLS support (configurable via ssl_verify parameter)

## Exported Functions

Public API surface defined in `NAMESPACE`:
- `authenticate_xnat()` - XNAT server authentication
- `list_projects()` - Query available projects
- `list_subjects()` - Query subjects within project
- `list_experiments()` - Query experiments for subject
- `list_scans()` - Query scans for experiment
- `search_projects()` - Search projects by substring
- `download_xnat_file()` - Download single file
- `download_xnat_files()` - Download multiple files
- `download_files()` - Download files for specific scan/experiment
- `download_subject()` - Download all data for a subject
- `download_all_subjects()` - Download all project data
- `initialize_config()` - Create credentials config file
- `load_credentials()` - Load credentials from config file
