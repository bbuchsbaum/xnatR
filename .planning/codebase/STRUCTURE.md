## Directory Purposes

**R/**
- Purpose: Source code for all package functions
- Contains: R functions exported to users via NAMESPACE
- Key files: All 10 .R files contain function definitions with Roxygen documentation comments

**man/**
- Purpose: Auto-generated R documentation from Roxygen comments in R/ files
- Contains: .Rd files for each exported function
- Key files: Created by `roxygen2::roxygenise()` during package build
- Note: Do NOT edit directly; regenerate from R/ source comments

**Root level**
- `DESCRIPTION`: Package metadata (name, version, author, license, dependencies)
- `NAMESPACE`: Exported function list (auto-generated from `@export` tags in R files)
- `xnatR.Rproj`: RStudio project configuration
- `.Rbuildignore`: Patterns to exclude from R package builds
- `.gitignore`: Patterns to exclude from version control

## Naming Conventions

**Files:**
- Lowercase with underscores: `authenticate.R`, `list_projects.R`, `download_files.R`
- One primary function per file (plus helper functions in download_files.R for related bulk operations)
- Pattern: `[action]_[resource].R` where action is verb (list, download, search) and resource is noun (projects, subjects, files)

**Functions:**
- Verb-noun pattern: `list_projects()`, `download_files()`, `search_projects()`, `authenticate_xnat()`
- Exported functions use underscores: `download_xnat_file()`, `download_all_subjects()`
- Private environment object: `xnatR_env` (all caps, single env object)
- Helper/internal functions: None explicitly marked private (R convention uses `.` prefix or lowercase, not followed here)

**Variables:**
- Parameter names: Lowercase with underscores: `project_id`, `subject_id`, `experiment_id`, `scan_id`, `dest_file`, `dest_dir`
- URL variables: Suffix `_url`: `projects_url`, `subjects_url`, `experiments_url`, `scans_url`, `base_url`
- Configuration variables: Prefix with type: `auth_header`, `ssl_config`, `content_json`, `content_parsed`

**Data Structures:**
- Data frames returned from queries: Named with singular noun: `projects_df`, `subjects_df`, `experiments_df`, `scans_df`
- Response objects: `response` (from httr::GET)
- Lists: `responses` (list of httr responses from bulk operations)

## Where to Add New Code

**New Feature (XNAT Listing/Query):**
- Primary code: `R/list_[resource].R` (e.g., `R/list_resources.R` for a new resource type)
- Follow pattern from `R/list_scans.R`:
  1. Validate all required parameters (project_id, subject_id, etc.)
  2. Check authentication state: `if (is.null(xnatR_env$auth_header)) stop("Not authenticated")`
  3. Construct URL using sprintf with URLencode
  4. Set SSL config from xnatR_env
  5. Execute httr::GET with Authorization header
  6. Parse JSON response with jsonlite::fromJSON
  7. Extract ResultSet$Result and convert to data frame
  8. Return data frame
- Export: Add `@export` tag to roxygen comment, then run `roxygen2::roxygenise()`

**New Feature (Download Variant):**
- Primary code: Add function to `R/download_files.R`
- Follow pattern from `download_subject()`:
  1. Validate parameters (project_id, subject_id, etc.)
  2. Check authentication: `if (is.null(xnatR_env$auth_header)) stop("Not authenticated")`
  3. Ensure dest_dir exists: `if (!dir.exists(dest_dir)) dir.create(dest_dir, recursive = TRUE)`
  4. List resources at current level (call appropriate list_* function)
  5. For each resource, construct download URL and call download_files() or httr::GET
  6. Return invisibly(responses) - list of httr response objects
- Export: Add `@export` tag to roxygen comment

**New Authentication Method:**
- Primary code: Extend `R/authenticate.R`
- Currently supports: Basic Auth (password) and Token-based Auth (API token as password)
- To add new method:
  1. Modify `authenticate_xnat()` to accept new parameter (e.g., `oauth_token`)
  2. Add conditional logic to construct appropriate auth header
  3. Store in xnatR_env for later use by API functions
  4. Test connection with GET request to verify

**New Configuration Option:**
- Primary code: Modify `R/config.R`
- Config file: `~/.xnatR_config.yml` (YAML format)
- To add option:
  1. Update template in `initialize_config()` (around line 18)
  2. Update `load_credentials()` to extract new field (around line 58)
  3. Store in xnatR_env if needed for session state
  4. Update relevant functions to use new configuration value

**Utilities/Helpers:**
- Shared helpers: Add to appropriate existing file or create `R/utils.R`
- Examples of candidates for helper extraction: URL construction, response parsing, error handling

## Special Directories

**man/**
- Purpose: R documentation files (.Rd format)
- Generated: Yes (auto-generated by roxygen2 from `@` tags in R/ source files)
- Committed: Yes (included in package distribution)
- Do NOT edit: Edit R/ source file roxygen comments instead, then run roxygen2::roxygenise()

**.Rproj.user/**
- Purpose: RStudio project user-specific settings and state
- Generated: Yes (by RStudio)
- Committed: No (typically in .gitignore)
- Do NOT edit: Automatically managed by RStudio

**R/**
- Purpose: R source code
- Generated: No (manually written)
- Committed: Yes (core package content)
- Style: Each file contains roxygen2 documentation comments starting with `#'`

- Committed: Yes (these documents help guide future development)
