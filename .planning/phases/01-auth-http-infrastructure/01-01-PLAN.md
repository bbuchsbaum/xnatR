    Create these internal functions:
    - `xnat_error(message, class, call = NULL, response = NULL)` -- base constructor that creates a condition object with classes `c(class, "xnat_error", "error", "condition")`, storing the message, response object, and HTTP status code (if response provided). Use `structure(class = ..., list(message = ..., status = ..., response = ..., call = ...))`.
    - `xnat_auth_error(response, message = NULL)` -- for 401 errors after re-auth fails. Default message: "Authentication failed - check credentials". Extract status from response.

<task type="auto">
  <name>Task 2: Create http-helpers.R centralized request wrapper</name>
  <files>R/http-helpers.R</files>
  <action>
    **R/http-helpers.R** (NEW): Create the internal `xnat_request()` function. This is the single point of entry for ALL HTTP calls in the package. Not exported -- internal use only.

    Function signature:
    ```r
    xnat_request <- function(method = c("GET", "POST", "PUT", "DELETE"),
                             path,
                             body = NULL,
                             query = list(),
                             accept = "application/json",
                             format = NULL,
                             max_retries = getOption("xnatR.retries", 3),
                             timeout = getOption("xnatR.timeout", 60),
                             auto_reauth = TRUE,
                             write_to = NULL,
                             ...)
    ```

    Implementation details:
    1. `method <- match.arg(method)`
    2. Check xnatR_env$base_url is set, stop with "Not authenticated. Call authenticate_xnat() first." if NULL.
    3. Build full URL: `paste0(sub("/+$", "", xnatR_env$base_url), "/", sub("^/+", "", path))`
    4. If `format` is provided (one of "json", "xml", "csv"), add it to the query list: `query$format <- format`. This handles HTTP-03.
    5. Build SSL config from xnatR_env$ssl_verify.
    6. Build auth: if xnatR_env$jsessionid is set, use `httr::set_cookies(JSESSIONID = xnatR_env$jsessionid)`. Otherwise fall back to `httr::add_headers(Authorization = xnatR_env$auth_header)`.
    7. Optionally add `httr::write_disk(write_to, overwrite = TRUE)` if write_to is not NULL (for downloads).
    8. Use `httr::RETRY()` with:
       - verb = method
       - url = full_url
       - config = ssl_config
       - auth cookie or header (from step 6)
       - httr::accept(accept)
       - httr::timeout(timeout)
       - query = query
       - body = body
       - times = max_retries
       - pause_base = 1, pause_cap = 60, pause_min = 1
       - terminate_on = c(400, 401, 403, 404) -- don't retry client errors
       - quiet = !getOption("xnatR.verbose", FALSE)
    9. Handle 401 with auto re-auth (if auto_reauth = TRUE):
       - If status 401, call `reauthenticate()` (defined below)
       - Retry the SAME request ONCE with times = 1 and auto_reauth = FALSE (prevent infinite loop)
       - If still 401, fall through to check_response which will raise xnat_auth_error
    10. Call `check_response(response)` from errors.R
    11. Return response invisibly

    Also create internal `reauthenticate()`:
    - Uses cached xnatR_env$username, xnatR_env$password or xnatR_env$token
    - If no cached credentials, stop with xnat_auth_error
    - Constructs Basic Auth header from cached creds
    - POSTs to /data/JSESSION to get new session
    - Updates xnatR_env$jsessionid and xnatR_env$auth_header
    - If verbose, message("Session refreshed")

    Also create internal helper `xnat_parse_response(response, as = c("json", "text", "raw"))`:
    - "json": parse with jsonlite::fromJSON(httr::content(response, as = "text", encoding = "UTF-8"), flatten = TRUE)
    - "text": httr::content(response, as = "text", encoding = "UTF-8")
    - "raw": httr::content(response, as = "raw")
    - This standardizes response parsing across the package.

    Important notes:
    - Do NOT use httr::handle() for cookie persistence yet -- just use JSESSIONID cookie header directly. The handle approach requires careful lifecycle management that's better handled when we rewrite authenticate_xnat() in Plan 02.
    - All functions are internal (no @export tags). Add roxygen2 @keywords internal tags.
    - Use xnatR_env$ for all state access.
  </action>
